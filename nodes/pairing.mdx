# 노드 페어링

노드(Node)는 OpenClaw Gateway에 연결된 **디바이스**입니다.
Mac, iPhone, iPad 등을 페어링하면 카메라, 위치, 화면 녹화 등을 원격으로 사용할 수 있습니다.

## 노드란?

```
[Gateway (서버)] ←→ [Node (디바이스)]
     ↑                    ↑
  AI 두뇌            물리적 세계
```

Gateway가 AI의 두뇌라면, 노드는 AI의 **눈과 귀**입니다.

<CardGroup cols={3}>
  <Card title="📱 iPhone/iPad" icon="mobile">
    카메라, 위치, 알림
  </Card>
  <Card title="💻 Mac" icon="laptop">
    화면 녹화, 명령 실행, 카메라
  </Card>
  <Card title="🖥️ 리모트 서버" icon="server">
    원격 명령 실행
  </Card>
</CardGroup>

## 페어링 방법

<Steps>
  <Step title="노드 호스트 설치 (Mac)">
```bash
# 노드 호스트 서비스 설치
openclaw node install

# 또는 포그라운드로 직접 실행
openclaw node run
```
  </Step>
  <Step title="페어링 요청 확인">
Gateway 측에서 대기 중인 요청을 확인합니다:
```bash
openclaw nodes list
```
  </Step>
  <Step title="페어링 승인">
```bash
openclaw nodes approve --node "my-macbook"
```
또는 AI에게 직접 요청:
```
"새 노드 페어링 요청 승인해줘"
```
  </Step>
  <Step title="연결 확인">
```bash
openclaw nodes describe --node "my-macbook"
```
노드의 기능(카메라, 위치 등)을 확인할 수 있습니다.
  </Step>
</Steps>

## iOS 앱 페어링

iPhone/iPad는 OpenClaw 앱을 통해 페어링합니다:

<Steps>
  <Step title="앱 설치">
    App Store에서 OpenClaw 설치
  </Step>
  <Step title="QR 코드 스캔">
    Gateway 대시보드의 QR 코드를 스캔하거나, 수동으로 Gateway URL 입력
  </Step>
  <Step title="권한 허용">
    카메라, 위치, 알림 권한을 허용합니다
  </Step>
  <Step title="승인">
    Gateway에서 페어링 승인
  </Step>
</Steps>

## 노드 관리

```bash
# 페어링된 노드 목록
openclaw nodes list

# 노드 상세 정보 (기능, 상태)
openclaw nodes describe --node "my-iphone"

# 노드에 알림 보내기 (macOS)
openclaw nodes notify --node "my-macbook" \
  --title "알림" --body "작업 완료!"

# 대기 중인 요청 거부
openclaw nodes reject --node "unknown-device"
```

## AI에서 노드 사용

AI는 `nodes` 도구를 통해 페어링된 디바이스에 접근합니다:

```
사용자: "거실 지금 어떤 상태야?"
→ AI: nodes.camera_snap(node="my-iphone", facing="back")
→ 사진 촬영 후 분석
→ "거실 불이 켜져 있고, 고양이가 소파에 있네요"
```

<Info>
노드 이름은 `TOOLS.md`에 기록해두면 AI가 어떤 디바이스인지 쉽게 파악합니다.
예: `my-iphone → 거실 테이블 위, 주로 CCTV 용도`
</Info>

## 보안

<Warning>
노드 페어링은 디바이스에 원격 접근 권한을 부여합니다.
알 수 없는 페어링 요청은 반드시 거부하세요.
</Warning>

- 페어링은 **수동 승인** 필수
- 각 노드별로 접근 가능한 기능이 다름
- Gateway 토큰으로 인증
