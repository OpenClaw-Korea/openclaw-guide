OpenClaw를 처음 시작하는 분을 위한 4주 가이드입니다.
매주 하나씩 익히면 자연스럽게 AI 어시스턴트와 함께하는 생활이 됩니다.

## 1주차: 설치와 첫 대화

**목표:** OpenClaw를 설치하고 기본 대화를 시작한다.

- [ ] OpenClaw 설치 (`npm install -g @anthropic/openclaw`)
- [ ] Gateway 시작 (`openclaw gateway start`)
- [ ] Telegram 봇 연결
- [ ] 첫 메시지 보내기: "안녕, 너 뭐할 수 있어?"

<Tip>
설치가 막히면 [설치 가이드](/getting-started/install)를 참고하세요.
대부분의 문제는 Node.js 버전이나 API 키 설정입니다.
</Tip>

## 2주차: 일상 활용

**목표:** 매일 1번 이상 OpenClaw에게 뭔가를 부탁한다.

활용 예시:
- "오늘 서울 날씨 알려줘"
- "이 영어 문장 자연스럽게 번역해줘"
- "Python에서 리스트 컴프리헨션 예시 보여줘"
- "이 에러 메시지 해석해줘: `TypeError: ...`"

핵심은 **습관**입니다. 뭔가 궁금하면 구글 대신 OpenClaw에게 먼저 물어보세요.

## 3주차: 파일과 작업

**목표:** 파일 읽기/쓰기, 간단한 명령 실행을 경험한다.

- [ ] "README.md 읽어봐" → 파일 읽기
- [ ] "회의록 정리해서 저장해줘" → 파일 쓰기
- [ ] "git status 확인해줘" → 명령 실행
- [ ] "이 폴더 구조 설명해줘" → 프로젝트 이해

<Info>
OpenClaw는 워크스페이스 폴더 내의 파일을 자유롭게 읽고 쓸 수 있습니다.
민감한 파일은 워크스페이스 밖에 보관하세요.
</Info>

## 4주차: 개인화

**목표:** AGENTS.md와 SOUL.md를 수정해서 나만의 어시스턴트로 만든다.

- [ ] `SOUL.md` 작성 — AI의 성격과 말투 정의
- [ ] `AGENTS.md` 수정 — 규칙과 워크플로우 추가
- [ ] `TOOLS.md`에 내 환경 정보 기록
- [ ] `USER.md` 작성 — 내 소개와 선호도

```markdown
# SOUL.md 예시
- 반말로 대화
- 답변은 간결하게
- 코드는 항상 TypeScript로
- 이모지 적당히 사용
```

## 체크포인트

4주가 끝나면 이 정도는 자연스러워야 합니다:

- ✅ 궁금한 건 OpenClaw에게 먼저 물어본다
- ✅ 간단한 파일 작업을 부탁할 수 있다
- ✅ AI의 성격과 규칙을 커스터마이징했다
- ✅ 하루에 최소 3-5번은 대화한다

완벽하게 마스터할 필요 없습니다.
다음 단계인 [중급자 로드맵](/roadmap/intermediate)으로 넘어가면서
자연스럽게 실력이 늡니다.
</Tip>
