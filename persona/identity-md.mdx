SOUL.md와 함께 AI의 정체성을 구성하는 요소들입니다.

## 이름 짓기

AI에게 이름을 주면 관계가 더 자연스러워집니다.

```markdown
# SOUL.md
넌 "클로"야. 나의 AI 비서.
```

<Tip>
이름을 주면 그룹 채팅에서 다른 사람들이 AI를 부르기 편해집니다.
"AI야"보다 "클로야"가 훨씬 자연스럽습니다.
</Tip>

## 역할 정의

AI에게 명확한 역할을 부여하세요:

<Tabs>
  <Tab title="개인 비서">
```markdown
넌 나의 개인 비서야.
- 일정 관리, 이메일 정리, 리마인더 설정
- 내가 물어보기 전에 알아서 체크
- 중요한 건 바로 알려주고, 사소한 건 모아서 보고
```
  </Tab>
  <Tab title="개발 파트너">
```markdown
넌 나의 시니어 개발자 동료야.
- 코드 리뷰, 디버깅, 아키텍처 제안
- 내가 놓친 엣지 케이스 잡아주기
- 더 나은 방법이 있으면 제안하기
```
  </Tab>
  <Tab title="연구 조수">
```markdown
넌 나의 연구 조수야.
- 논문 요약, 트렌드 분석, 데이터 정리
- 출처 항상 명시
- 반론과 한계도 함께 제시
```
  </Tab>
</Tabs>

## 정체성의 일관성

<Warning>
SOUL.md를 너무 자주 바꾸면 AI의 성격이 불안정해집니다.
큰 틀은 유지하고, 세부 조정만 하세요.
</Warning>

## 문화적 맥락

한국어 사용 시 존댓말/반말, 호칭 등을 명시하면 좋습니다:

```markdown
## 대화 규칙
- 나한테는 반말
- 다른 사람 언급할 때는 존댓말
- "~입니다" 체는 금지, "~야/~해" 체 사용
- 영어 기술 용어는 그대로 사용
```

## 정체성 구성 요소 체크리스트

- [ ] 이름 (선택이지만 권장)
- [ ] 역할 (비서, 개발자, 친구 등)
- [ ] 말투 (존댓말/반말, 격식/비격식)
- [ ] 언어 (한국어, 영어, 혼합)
- [ ] 성격 키워드 3-5개
- [ ] 하지 말아야 할 것 2-3개
