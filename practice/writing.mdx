OpenClaw는 파일 시스템에 직접 접근할 수 있어서
"블로그 글 써줘"라고 하면 실제 파일로 저장까지 완료합니다.

## 기본 사용법

### 블로그 작성

```
"Next.js App Router 가이드 블로그 글 써줘"
"어제 회의 내용으로 회고 포스트 만들어줘"
```

OpenClaw 동작:
```
1. 주제 파악 및 구조 설계
2. 필요시 web_search로 최신 정보 보충
3. 마크다운으로 초안 작성
4. 파일로 저장 (예: blog/nextjs-app-router.md)
```

### 문서 작성

```
"README.md 작성해줘 — Python CLI 프로젝트야"
"API 문서 템플릿 만들어줘"
"이 코드에 JSDoc 주석 달아줘"
```

<Tip>
기존 파일을 읽고 스타일을 맞출 수 있습니다.
"기존 README 스타일에 맞춰서 CONTRIBUTING.md 써줘"처럼 요청하세요.
</Tip>

### 번역

```
"이 README를 한국어로 번역해줘"
"영어로 이메일 초안 작성해줘 — 미팅 일정 조율 내용"
```

```bash
# 파일 번역
"docs/guide.md를 영어로 번역해서 docs/guide-en.md로 저장해줘"

# 클립보드 번역 (macOS)
"클립보드에 있는 텍스트 일본어로 번역해줘"
```

## 실전 패턴

### 반복 글쓰기 워크플로우

```
사용자: "주간 개발 회고 써줘"

OpenClaw 동작:
1. memory/ 폴더에서 이번 주 기록 확인
2. git log로 이번 주 커밋 히스토리 수집
3. 성과, 배운 점, 다음 주 계획으로 구조화
4. blog/weekly/2025-02-10.md로 저장
```

### 교정 및 개선

```
"이 글 문법 체크하고 더 자연스럽게 다듬어줘"
"너무 딱딱한데 좀 캐주얼하게 바꿔줘"
"영어 원어민이 쓴 것처럼 다듬어줘"
```

<Info>
원본 파일을 직접 수정하거나, 새 파일로 저장할 수 있습니다.
"원본은 건드리지 말고 새 파일로 저장해줘"라고 명시하세요.
</Info>

### 시리즈 콘텐츠

```
사용자: "OpenClaw 튜토리얼 시리즈 기획해줘 — 5편짜리"

OpenClaw 동작:
1. 전체 시리즈 아웃라인 설계
2. 각 편의 제목, 핵심 내용, 예상 분량 정리
3. 1편부터 순차적으로 작성
4. 이전 편의 맥락을 유지하며 연결
```

## 자동화

### 일일 TIL 자동 생성

```bash
openclaw cron add --name "daily-til" \
  --cron "0 22 * * *" \
  --message "오늘 memory 파일과 git log 보고 TIL 작성해서 til/ 폴더에 저장해줘" \
  --session isolated
```

<Warning>
외부 공개용 글(블로그 배포, SNS 포스팅)은 반드시 확인 후 발행하세요.
AGENTS.md에 "글 발행 전 확인" 규칙을 추가하는 것을 권장합니다.
</Warning>
