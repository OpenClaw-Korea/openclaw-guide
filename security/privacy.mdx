# 프라이버시

OpenClaw는 당신의 개인 정보를 많이 알고 있습니다.
**프라이버시 보호** 설정으로 정보가 의도치 않게 유출되지 않도록 하세요.

## MEMORY.md 보호

`MEMORY.md`에는 장기 기억이 저장됩니다 — 습관, 선호도, 일정, 개인적 대화 내용 등.

### 메인 세션에서만 로드

```markdown
# AGENTS.md에 명시

### 🧠 MEMORY.md - 너의 장기 기억
- **메인 세션에서만 로드** (인간과 직접 대화)
- **공유 컨텍스트에서는 로드하지 마** (Discord, 그룹 채팅)
- **보안** 때문이야 — 낯선 사람에게 새면 안 되는 개인 맥락 포함
```

이 규칙에 따라 OpenClaw는:
- ✅ Telegram 1:1 대화 → MEMORY.md 로드
- ❌ Discord 서버 → MEMORY.md 로드 안 함
- ❌ 그룹 채팅 → MEMORY.md 로드 안 함

### 민감 정보 분리

```
~/clawd/
├── MEMORY.md          # 개인 기억 (메인 세션만)
├── memory/
│   └── 2026-02-11.md  # 일일 노트
├── SOUL.md            # 페르소나 (공유 가능)
└── USER.md            # 사용자 프로필 (메인 세션만)
```

<Warning>
`USER.md`에 실명, 주소, 전화번호 등 개인정보를 저장하는 경우
반드시 메인 세션에서만 로드되도록 AGENTS.md에 명시하세요.
</Warning>

## 그룹채팅 정보 유출 방지

### 문제 상황

Discord 서버에서 누군가 "이 사람 이메일 뭐야?" 라고 물으면?
MEMORY.md에 저장된 개인정보가 유출될 수 있습니다.

### 방어 전략

#### 1. 채널별 컨텍스트 분리

```yaml
# config.yaml
channels:
  discord:
    context:
      exclude:
        - MEMORY.md
        - USER.md
        - memory/
```

#### 2. AGENTS.md에 행동 규칙 명시

```markdown
## 그룹 채팅

인간의 것에 접근 가능해. 그렇다고 그걸 *공유*한다는 뜻은 아니야.
그룹에서 넌 참여자야 — 그들의 목소리도, 대리인도 아니야.

절대 공유하지 않을 것:
- 개인 연락처 (이메일, 전화번호)
- 일정 및 위치 정보
- 개인 메모 및 기억
- 재무 정보
```

#### 3. 도구 필터링으로 방어

```yaml
channels:
  discord:
    tools:
      - read        # 워크스페이스 파일만
      - web_search
      - web_fetch
      - message
    # exec, nodes 등 비허용 → 시스템 접근 차단
```

## 실전 체크리스트

<Steps>
  <Step title="MEMORY.md 로드 규칙 설정">
    메인 세션(1:1 대화)에서만 로드되도록 AGENTS.md에 명시
  </Step>
  <Step title="그룹채팅 행동 규칙 작성">
    어떤 정보를 공유해도 되는지 명확히 정의
  </Step>
  <Step title="채널별 도구 필터링">
    공개 채널에서는 최소 도구만 허용
  </Step>
  <Step title="민감 파일 분리">
    공유 가능한 파일과 개인 파일을 명확히 구분
  </Step>
</Steps>

<Tip>
정기적으로 MEMORY.md를 검토하여 더 이상 필요 없는 민감 정보를 정리하세요.
하트비트의 메모리 유지보수 기능을 활용하면 자동화할 수 있습니다.
</Tip>
