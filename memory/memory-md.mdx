# MEMORY.md — 장기 기억

MEMORY.md는 AI의 **정제된 장기 기억**입니다.
일일 노트(memory/YYYY-MM-DD.md)가 날 것의 일기라면, MEMORY.md는 핵심만 추려낸 지혜입니다.

## 동작 방식

```
매 세션 시작 시 (메인 세션에서만):
1. SOUL.md 읽기
2. USER.md 읽기
3. 오늘/어제 일일 노트 읽기
4. MEMORY.md 읽기 ← 장기 기억
```

<Warning>
**보안 중요**: MEMORY.md는 메인 세션(1:1 DM)에서만 로드됩니다.
그룹 채팅이나 공유 채널에서는 절대 로드되지 않습니다.
개인적인 정보가 타인에게 노출되지 않도록 하기 위함입니다.
</Warning>

## 구조 예시

```markdown
# MEMORY.md

## 중요한 결정
- 2026-01-15: 프로젝트 A를 React에서 Next.js로 마이그레이션 결정
- 2026-01-20: 이메일 자동 분류 규칙 합의 (긴급/일반/무시)

## 배운 교훈
- exec 정책은 allowlist로 시작하는 게 안전하다
- 하트비트 간격 15분은 너무 짧다 → 30분이 적당
- WhatsApp은 별도 번호 사용이 좋다

## 진행 중인 프로젝트
- 블로그 마이그레이션: 70% 완료, 이미지 처리 남음
- 홈 자동화: HomeKit 연동 테스트 중

## 선호사항 (대화에서 발견)
- 아침에 브리핑할 때 날씨 먼저
- 코드 리뷰 시 TypeScript strict mode 선호
- 주말에는 업무 알림 최소화
```

## AI가 자동으로 관리

AI는 하트비트 시간에 주기적으로:

1. 최근 일일 노트들을 검토
2. 장기 보관할 가치가 있는 정보를 식별
3. MEMORY.md를 정제된 내용으로 업데이트
4. 더 이상 관련 없는 오래된 정보 제거

<Tip>
"이거 기억해" 또는 "이건 중요해"라고 말하면 AI가 즉시 기록합니다.
반대로 "이거 잊어도 돼"라고 하면 삭제합니다.
</Tip>

## MEMORY.md vs 일일 노트

| MEMORY.md | 일일 노트 (memory/YYYY-MM-DD.md) |
|-----------|----------------------------------|
| 정제된 핵심 | 날 것의 기록 |
| 수동+자동 업데이트 | 매 세션 자동 기록 |
| 항상 로드 (메인) | 오늘+어제만 로드 |
| 오래 유지 | 시간이 지나면 참조만 |

## 수동 관리

AI에게 직접 지시할 수도 있습니다:

```
"MEMORY.md에서 프로젝트 A 관련 내용 삭제해줘"
"최근 일주일 기록 정리해서 MEMORY.md 업데이트해줘"
"MEMORY.md 보여줘"
```
